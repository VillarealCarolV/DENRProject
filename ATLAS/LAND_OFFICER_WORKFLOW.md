# ATLAS System - Land Officer Workflow Documentation

## Overview
This document provides comprehensive developer instructions for the **Land Officer Assessment Workflow** in the ATLAS (Automated Tracking of Land Application System). The workflow allows Land Management Officers to review, assess, and finalize applications submitted by Records Officers.

---

## User Roles & Permissions

### Role: `records_officer`
- **Responsibility**: Input new applications
- **Permissions**:
  - Create applications using the Master Intake Form
  - View all applications
  - Cannot edit applications (read-only access)

### Role: `land_management_officer`
- **Responsibility**: Physical verification and assessment
- **Permissions**:
  - View all applications
  - Edit/Assess pending applications
  - Make final decisions (Approve/Reject)
  - Update application status and remarks
  - Cannot create new applications

### Role: `admin`
- **Responsibility**: System administration
- **Permissions**: Full access to all features

---

## Application Status Flow

```
Pending → In Process → Approved / Rejected
           ↑
      (Records Officer input)
           ↓
    (Land Officer assesses)
           ↓
    Final Status
```

### Status Definitions:
- **Pending**: Initial status when application is created (Auto-generated by system)
- **In Process**: Land Officer is actively assessing the application
- **Approved**: Land Officer approved the application after physical verification
- **Rejected**: Land Officer rejected the application (with reasons in remarks)

---

## Land Officer Assessment Workflow

### Step 1: Lot Classification
The Land Officer must physically verify the land and determine:

**Option A: Existing Lot**
- No subdivisions
- Uses current cadastral data
- Skips subdivision details
- Quick approval process

**Option B: Subdivision**
- Lot is divided from the mother lot
- Requires new lot number
- Requires subdivided area measurement
- System automatically calculates remaining area of mother lot

### Step 2: Subdivision Details (If Applicable)
If "Subdivision" is selected:

**Required Fields:**
- **New Lot Number**: Identifier for the new subdivided lot (e.g., "001-A", "2B-North")
- **Subdivided Area**: Area of the new lot in square meters (must be less than total mother lot area)

**Automatic Calculation:**
```
Remaining Area = Mother Lot Total Area - Subdivided Area
```

The system validates that the subdivided area doesn't exceed the mother lot's total area.

### Step 3: Final Decision & Remarks
The Land Officer must:

1. **Select Status**:
   - "In Process" - Assessment ongoing, not finalized
   - "Approved" - Application approved after verification
   - "Rejected" - Application rejected with reasons

2. **Add Remarks** (Required, minimum 10 characters):
   - Approval: "Approved based on Cadastral survey. All boundaries verified. Ready for patent processing."
   - Rejection: "Rejected due to conflicting boundary data. Physical survey required before resubmission."
   - Assessment: "Subdivision verified. New lot area conforms to requirements. Remaining area adequate for future development."

3. **Patent Details** (Optional):
   - Patent Type: e.g., "Agricultural Free Patent", "Residential"
   - Patent Details: Additional patent information

4. **Save**: Submit the assessment

## Database Schema

### New Fields in `applications` Table

| Field | Type | Nullable | Description |
|-------|------|----------|-------------|
| `lot_type` | ENUM('existing_lot', 'subdivision') | Yes | Classification of the lot |
| `new_lot_number` | VARCHAR(255) | Yes | New lot identifier (for subdivisions) |
| `subdivided_area` | DECIMAL(10,2) | Yes | Area of subdivided lot in sqm |
| `remaining_area` | DECIMAL(10,2) | Yes | Calculated remaining area of mother lot |
| `land_officer_remarks` | TEXT | Yes | Official remarks from Land Officer |
| `land_officer_id` | BIGINT (FK to users) | Yes | ID of assessing Land Officer |
| `assessed_at` | TIMESTAMP | Yes | Date/time of assessment completion |

### Related Tables:
- `land_records`: Contains mother lot information (survey_no, total_area, location)
- `applicants`: Contains applicant information
- `status_histories`: Audit trail of all status changes
- `users`: Contains Land Officer user information

## Validation Rules

### Lot Type Assessment
```php
'lot_type' => 'required|in:existing_lot,subdivision'
```

### Subdivision Fields (Conditional)
```php
'new_lot_number' => 'nullable|required_if:lot_type,subdivision|string|max:255'
'subdivided_area' => 'nullable|required_if:lot_type,subdivision|numeric|min:0.01'
```
- Only required if lot_type is "subdivision"
- Subdivided area must be positive and less than mother lot total
- System validates that subdivided_area < total_area

### Status Update
```php
'status' => 'required|in:In Process,Approved,Rejected'
```
- Must select one of three options
- Cannot change back to "Pending"

### Remarks
```php
'land_officer_remarks' => 'required|string|min:10'
```
- Minimum 10 characters required
- Should explain decision rationale
- Becomes part of audit trail

**Layout:**
1. Application Summary Card
   - Applicant Information (read-only)
   - Land Record Information (read-only)
   - Mother lot total area display

2. Assessment Form with 3 Steps:
   - **Step 1**: Lot Classification (Radio buttons)
   - **Step 2**: Subdivision Details (Conditional fields)
   - **Step 3**: Final Decision & Remarks

**Features:**
- Conditional visibility: Subdivision fields show/hide based on lot type selection
- Real-time calculation: Remaining area updates as subdivided area changes
- Validation feedback: Form highlights errors
- Calculation validation: Prevents subdivided area from exceeding mother lot
- Status history timeline: Shows previous updates and audit trail

**JavaScript:**
```javascript
// Toggle subdivision fields visibility based on lot type selection
function toggleSubdivisionFields() {
  if (lotType === 'subdivision') {
    subdivisionFields.show();
  } else {
    subdivisionFields.hide();
  }
}

// Calculate remaining area in real-time
function calculateRemainingArea() {
  remainingArea = motherLotTotal - subdividedArea;
  if (remainingArea < 0) {
    showValidationError("Subdivided area exceeds mother lot total");
  }
}
```

## Controller Methods

### ApplicationController::edit($id)
```php
public function edit(string $id)
{
    // Authorization check
    if (Auth::user()->role !== 'land_management_officer') {
        abort(403, 'Only Land Officers can assess applications');
    }
    
    // Load application with relationships
    $application = Application::with([
        'applicant', 
        'landRecord', 
        'statusHistories'
    ])->findOrFail($id);
    
    return view('applications.edit', compact('application'));
}
```

### ApplicationController::update(Request $request, Application $application)
```php
public function update(Request $request, Application $application)
{
    // Authorization check
    if (Auth::user()->role !== 'land_management_officer') {
        abort(403, 'Only Land Officers can update applications');
    }
    
    // Validate all inputs including conditional fields
    $validated = $request->validate([
        'lot_type' => 'required|in:existing_lot,subdivision',
        'new_lot_number' => 'nullable|required_if:lot_type,subdivision',
        'subdivided_area' => 'nullable|required_if:lot_type,subdivision|numeric|min:0.01',
        'status' => 'required|in:In Process,Approved,Rejected',
        'land_officer_remarks' => 'required|string|min:10',
        'patent_details' => 'nullable|string',
        'patent_type' => 'nullable|string',
    ]);
    
    // Handle subdivision logic
    if ($validated['lot_type'] === 'subdivision') {
        $remaining = $application->landRecord->total_area - $validated['subdivided_area'];
        $application->remaining_area = $remaining;
    }
    
    // Update application with new assessment data
    $application->update([
        'lot_type' => $validated['lot_type'],
        'new_lot_number' => $validated['new_lot_number'] ?? null,
        'subdivided_area' => $validated['subdivided_area'] ?? null,
        'remaining_area' => $remaining ?? null,
        'status' => $validated['status'],
        'land_officer_remarks' => $validated['land_officer_remarks'],
        'land_officer_id' => Auth::id(),
        'assessed_at' => now(),
    ]);
    
    // Create audit trail entry
    StatusHistory::create([
        'application_id' => $application->id,
        'status' => $validated['status'],
        'remarks' => $validated['land_officer_remarks'],
        'updated_by' => Auth::user()->name,
    ]);
    
    return redirect()->route('applications.show', $application)
        ->with('success', 'Assessment completed successfully!');
}
```

## Notifications System

### Notification Types:

1. **New Application Created**
   - Triggered: When Records Officer creates application
   - Recipients: Land Officers
   - Message: "New application {tracking_no} awaiting assessment"

2. **Application Approved**
   - Triggered: When Land Officer approves
   - Recipients: Records Officer (and possibly admin)
   - Message: "Application {tracking_no} has been approved"

3. **Application Rejected**
   - Triggered: When Land Officer rejects
   - Recipients: Records Officer
   - Message: "Application {tracking_no} has been rejected. Review remarks."

### Notification Modal
When clicking a notification, a modal displays:
- Full notification message
- Application tracking number
- Relevant details (applicant, lot type, status)
- Action buttons (View, Dismiss)

```php
// Accessed via: Auth::user()->notifications
```

## Error Handling & Validation

### Common Errors & Solutions:

**Error:** "Unauthorized: Only Land Officers can assess applications"
- **Cause**: User doesn't have `land_management_officer` role
- **Solution**: Update user role in database
```sql
UPDATE users SET role = 'land_management_officer' WHERE id = ?;
```

**Error:** "Subdivided area must be less than mother lot total"
- **Cause**: Input subdivided_area ≥ total_area
- **Solution**: Verify measurement and enter value less than mother lot

**Error:** "Remarks must be at least 10 characters"
- **Cause**: Remarks field has insufficient detail
- **Solution**: Provide more specific assessment remarks (minimum 10 chars)

**Error:** "New lot number is required for subdivisions"
- **Cause**: Selected subdivision but didn't enter new lot number
- **Solution**: Enter valid new lot identifier (e.g., "001-A")

## Performance Considerations

1. **Eager Loading**: Use `with()` to load related data
   - ApplicationController loads `['applicant', 'landRecord', 'statusHistories']`
   - Prevents N+1 query problems

2. **Indexing**: Recommend adding indices:
   ```sql
   ALTER TABLE applications ADD INDEX idx_lot_type (lot_type);
   ALTER TABLE applications ADD INDEX idx_land_officer_id (land_officer_id);
   ALTER TABLE applications ADD INDEX idx_assessed_at (assessed_at);
   ```

3. **Pagination**: Notifications use pagination to prevent memory overload
